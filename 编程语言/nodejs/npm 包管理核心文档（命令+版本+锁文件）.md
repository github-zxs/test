
# 一、npm 包增删改查核心命令

以下为最常用、最实用的 npm 包管理命令，涵盖项目依赖、全局依赖、查看、更新、删除全场景，直接复制即可使用。

## （一）增（安装包）

- **安装到项目依赖（生产环境，默认）**：打包上线会用到的包（如 vue、react、axios） `npm install 包名` `# 简写` `npm i 包名`
    
- **安装到开发依赖（仅开发用）**：打包不需要，仅开发时使用（如 webpack、vite、eslint） `npm install 包名 --save-dev` `# 简写` `npm i 包名 -D`
    
- **全局安装（电脑任意地方可用）**：工具类命令（如 vue-cli、create-react-app、nodemon） `npm install 包名 -g`
    
- **安装指定版本**： `npm i 包名@版本号` `# 示例` `npm i axios@1.6.0`
    
- **安装 package.json 里所有依赖**：克隆别人项目后必用 `npm install` `# 简写` `npm i`
    

## （二）删（卸载包）

- **卸载项目中的包**： `npm uninstall 包名` `# 简写` `npm un 包名`
    
- **卸载全局包**： `npm uninstall 包名 -g`
    

## （三）改（更新、升级包）

- **更新单个包到最新版**： `npm update 包名`
    
- **更新所有可更新的包**： `npm update`
    
- **查看哪些包可以更新**： `npm outdated`
    

## （四）查（查看包信息）

- **查看项目已安装的包**： `npm list` `# 只看顶层依赖（不看嵌套，最常用）` `npm list --depth=0`
    
- **查看全局安装的包**： `npm list -g --depth=0`
    
- **查看包的远程最新版本**： `npm view 包名 version` `# 查看包所有历史版本` `npm view 包名 versions`
    
- **查看包详情**： `npm info 包名`
    

## （五）常用快捷命令汇总

| 功能     | 命令                                                          |
| ------ | ----------------------------------------------------------- |
| 安装包    | `npm i 包名`                                                  |
| 安装开发依赖 | `npm i 包名 -D`                                               |
| 全局安装   | `npm i 包名 -g`                                               |
| 卸载包    | `npm un 包名`                                                 |
| 卸载全局包  | `npm un 包名 -g`                                              |
| 更新包    | `npm update 包名`,npm update -g @anthropic-ai/claude-code<br> |

| 安装所有依赖 | `npm i`                                                     |

# 二、package.json 中 ^5.0.0 版本号解释

在 package.json 中出现的 `^5.0.0`，是 npm 语义化版本（SemVer）的标准写法，核心用于规范依赖包的更新范围，兼顾灵活性和稳定性。

## （一）基础：版本号 5.0.0 的构成

npm 语义化版本遵循`主版本号.次版本号.修订号` 的格式，即：

```plain
5 . 0 . 0
↑   ↑   ↑
主  次  修订
```

- **主版本号（5）**：重大更新，可能包含不兼容旧代码的修改，升级后需检查项目兼容性。
    
- **次版本号（0）**：功能新增，保持向下兼容，不会影响现有代码运行。
    
- **修订号（0）**：小版本更新，仅修复 bug，完全兼容旧版本，无功能变更。
    

## （二）^ 符号的含义

**^（脱字符）** 是 npm 中最常用的版本范围符号，其规则为：**允许安装 ≥5.0.0 且 <6.0.0 的最新版本，不升级主版本号，仅升级次版本号和修订号**。

### ^5.0.0 允许的版本范围

- ✅ 允许安装（所有 5.x.x 版本）：5.0.1、5.1.0、5.2.3、5.9.9 等。
    
- ❌ 不允许安装（主版本号变更）：6.0.0、7.0.0 等。
    

## （三）使用 ^ 符号的优势

- 自动获取依赖包的 bug 修复和新增功能，无需手动修改版本号。
    
- 避免主版本号升级带来的不兼容问题，保证项目稳定运行。
    
- 平衡“依赖更新”和“项目稳定”，是 npm 最推荐的版本写法。
    

## （四）常见版本符号对比

|写法|含义|允许更新范围|
|---|---|---|
|`"5.0.0"`|固定版本|只能安装 5.0.0，无任何更新空间|
|`"^5.0.0"`|主版本固定，次版/修订版可更新|5.0.0 ~ 5.9.9|
|`"~5.0.0"`|主版/次版固定，仅修订版可更新|5.0.0 ~ 5.0.9|
|`">=5.0.0"`|大于等于指定版本|5.0.0 及以上所有版本（含 6.0.0+）|

## （五）实际使用场景

当你在 package.json 中写入：

```json
"electron-vite": "^5.0.0"
```

执行 `npm install` 时，npm 会自动安装当前 5.x.x 系列中的最新稳定版（如 5.1.0、5.2.0），但绝不会安装 6.0.0 及以上版本，避免不兼容问题。

# 三、package-lock.json 详解（与 ^5.0.0 关联）

package-lock.json 是 npm 自动生成的版本锁定文件，核心作用是解决 `^5.0.0` 带来的“版本不固定”问题，保证所有环境安装的依赖版本完全一致。

## （一）核心定位：愿望清单 vs 实际清单

一句话理清两个文件的关系：**package.json 是“愿望清单”（定更新规则），package-lock.json 是“实际安装清单”（锁具体版本）**。

- package.json：只规定规则（如 `^5.0.0`），不记录实际安装的具体版本。
    
- package-lock.json：记录每次 `npm install` 实际安装的依赖版本（如 5.1.0），并锁定该版本。
    

## （二）package-lock.json 的工作原理

当你执行 `npm install` 时，npm 会按以下步骤执行：

1. 读取 package.json 中的版本规则（如 `^5.0.0`）。
    
2. 查询当前符合规则的最新版本（如 5.1.0），并安装该版本。
    
3. 将实际安装的版本（5.1.0）、依赖路径、下载地址等信息，写入 package-lock.json 并锁定。
    

此后，无论谁、在什么时间、什么电脑上执行 `npm install`，都会优先读取 package-lock.json 中的锁定版本，安装 5.1.0，不会随时间推移自动升级到 5.2.0 等更高版本。

## （三）两个文件的核心区别

|文件|作用|版本会不会变？|
|---|---|---|
|package.json|定义依赖的版本规则（如 ^5.0.0）|不会自动变，需手动修改|
|package-lock.json|锁定实际安装的具体版本|仅在主动更新依赖时才会变|

## （四）真实示例

1. 你在 package.json 中写入 `"electron-vite": "^5.0.0"`，第一次执行 `npm install` 时，当前最新的 5.x.x 版本是 5.0.0，此时 lock 文件锁定 5.0.0。
    
2. 一个月后，electron-vite 发布 5.2.0 版本（符合 ^5.0.0 规则），你再次执行 `npm install`，依然会安装 5.0.0（因为 lock 文件锁定）。
    
3. 若想升级到 5.2.0，需主动执行`npm update electron-vite`，此时 lock 文件会更新为 5.2.0，后续安装都会使用该版本。
    

## （五）为什么必须提交 package-lock.json 到 Git？

核心目的是 **保证团队所有成员、开发环境、测试环境、生产服务器的依赖版本完全一致**，避免因版本差异导致的诡异 bug、运行异常。

若不提交 lock 文件，可能出现：A 电脑安装 5.0.0，B 电脑安装 5.2.0，服务器安装 5.1.0，导致项目在不同环境中运行效果不一致，增加调试成本。

## （六）终极总结

1. `^5.0.0`：主版本号不变，允许自动更新次版本号和修订号，兼顾灵活性和稳定性。
    
2. package-lock.json：锁定实际安装的依赖版本，确保所有环境安装的版本一致，解决“版本不固定”问题。
    
3. `npm install` 优先遵循 package-lock.json 的锁定版本，而非 package.json 的规则。
    
4. 一句话记忆：package.json 定规则，package-lock.json 锁版本。
