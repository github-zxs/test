在 NestJS 中，多个全局守卫的默认执行顺序由**注册顺序**决定（先注册先执行），但通过 `@SetMetadata()` 自定义元数据 + 反射器（`Reflector`）读取元数据，可实现**精细化的优先级排序**。以下是完整实现方案，包含「定义优先级元数据」「全局守卫排序逻辑」「多守卫优先级控制」全流程：

### 核心思路

1. 定义「优先级元数据装饰器」，为不同守卫标记优先级数值（数值越大，执行优先级越高）。
2. 全局守卫中通过 `Reflector` 读取自身优先级，按数值排序后执行。
3. 结合 Nest 生命周期，统一管理所有全局守卫的执行顺序。

## 步骤 1：定义优先级元数据装饰器

先创建装饰器，用于给守卫标记优先级（数值越大，执行越靠前）：

typescript

运行

```typescript
// src/decorators/guard-priority.decorator.ts
import { SetMetadata } from '@nestjs/common';

// 定义元数据 key
export const [GUARD](NestJS%20全局守卫排序：框架原生、自定义容器与依赖注入.md)_PRIORITY_KEY = 'guardPriority';

/**
 * 守卫优先级装饰器
 * @param priority 优先级数值（默认 0，数值越大执行越靠前）
 */
export const GuardPriority = (priority = 0) => 
  SetMetadata(GUARD_PRIORITY_KEY, priority);
```

## 步骤 2：定义带优先级的全局守卫

创建多个全局守卫，用 `@GuardPriority()` 标记优先级：

## 步骤 3：实现全局守卫排序逻辑

### 方案 1：统一注册并排序（推荐）

在根模块中，通过 `useFactory` 动态读取所有守卫的优先级，排序后注册为全局守卫：

typescript

运行

```typescript
// src/app.module.ts
import { Module } from '@nestjs/common';
import { APP_GUARD, Reflector } from '@nestjs/core';
import { Guard1 } from './guards/guard1.guard';
import { Guard2 } from './guards/guard2.guard';
import { Guard3 } from './guards/guard3.guard';
import { GUARD_PRIORITY_KEY } from './decorators/guard-priority.decorator';

@Module({
  providers: [
    // 注册所有全局守卫
    Guard1,
    Guard2,
    Guard3,
    // 统一排序并设置全局守卫
    {
      provide: APP_GUARD,
      useFactory: (reflector: Reflector) => {
        // 1. 获取所有守卫实例（需先注册为 providers）
        const guards = [new Guard1(reflector), new Guard2(reflector), new Guard3(reflector)];
        
        // 2. 按优先级排序（数值大的排前面）
        const sortedGuards = guards.sort((a, b) => {
          const priorityA = reflector.get(GUARD_PRIORITY_KEY, a.constructor) || 0;
          const priorityB = reflector.get(GUARD_PRIORITY_KEY, b.constructor) || 0;
          return priorityB - priorityA;
        });

        // 3. 返回排序后的守卫数组（Nest 会按数组顺序执行）
        return sortedGuards;
      },
      inject: [Reflector], // 注入反射器
    },
  ],
})
export class AppModule {}
```

### 方案 2：自定义守卫容器（进阶）

若需更灵活的控制（如动态增减守卫），可创建「守卫管理器」，统一管理排序和执行：

typescript

运行

```typescript
// src/guards/guard-manager.guard.ts
import { CanActivate, ExecutionContext, Injectable, Reflector } from '@nestjs/common';
import { Guard1 } from './guard1.guard';
import { Guard2 } from './guard2.guard';
import { Guard3 } from './guard3.guard';
import { GUARD_PRIORITY_KEY } from '../decorators/guard-priority.decorator';

@Injectable()
export class GuardManager implements CanActivate {
  // 所有待执行的守卫
  private readonly guards = [new Guard1(this.reflector), new Guard2(this.reflector), new Guard3(this.reflector)];

  constructor(private readonly reflector: Reflector) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    // 1. 按优先级排序守卫
    const sortedGuards = this.guards.sort((a, b) => {
      const priorityA = this.reflector.get(GUARD_PRIORITY_KEY, a.constructor) || 0;
      const priorityB = this.reflector.get(GUARD_PRIORITY_KEY, b.constructor) || 0;
      return priorityB - priorityA;
    });

    // 2. 依次执行所有守卫（只要有一个返回 false，整体校验失败）
    for (const guard of sortedGuards) {
      const result = await guard.canActivate(context);
      if (!result) return false;
    }

    return true;
  }
}

// 根模块注册
@Module({
  providers: [
    {
      provide: APP_GUARD,
      useClass: GuardManager,
    },
    Reflector, // 注入反射器
  ],
})
export class AppModule {}
```

## 进阶：路由级别的优先级覆盖

若需对**特定路由**调整守卫优先级（而非全局），可结合路由元数据：

typescript

运行

```typescript
// 定义路由优先级装饰器
export const ROUTE_GUARD_PRIORITY = 'routeGuardPriority';
export const RouteGuardPriority = (priority: number[]) => 
  SetMetadata(ROUTE_GUARD_PRIORITY, priority);

// 控制器中使用
@Controller()
export class AppController {
  @Get('special')
  // 为该路由指定守卫执行顺序：Guard3 → Guard1 → Guard2
  @RouteGuardPriority([0, 10, 5])
  specialRoute() {
    return '特殊路由，守卫顺序自定义';
  }
}

// 在 GuardManager 中适配路由优先级
async canActivate(context: ExecutionContext): Promise<boolean> {
  // 读取路由级别的优先级配置
  const routePriority = this.reflector.get<number[]>(
    ROUTE_GUARD_PRIORITY,
    context.getHandler(),
  );

  let sortedGuards = [...this.guards];
  if (routePriority) {
    // 按路由指定的优先级排序
    sortedGuards = sortedGuards.sort((a, b) => {
      const priorityA = this.reflector.get(GUARD_PRIORITY_KEY, a.constructor) || 0;
      const priorityB = this.reflector.get(GUARD_PRIORITY_KEY, b.constructor) || 0;
      const indexA = routePriority.indexOf(priorityA);
      const indexB = routePriority.indexOf(priorityB);
      return indexA - indexB;
    });
  } else {
    // 全局默认排序
    sortedGuards = sortedGuards.sort((a, b) => {
      const priorityA = this.reflector.get(GUARD_PRIORITY_KEY, a.constructor) || 0;
      const priorityB = this.reflector.get(GUARD_PRIORITY_KEY, b.constructor) || 0;
      return priorityB - priorityA;
    });
  }

  // 执行排序后的守卫
  for (const guard of sortedGuards) {
    const result = await guard.canActivate(context);
    if (!result) return false;
  }

  return true;
}
```

## 关键注意事项

1. **元数据读取范围**：`reflector.get()` 需指定守卫的 `constructor`（而非实例），因为元数据挂载在类上。
2. **异步守卫兼容**：若守卫的 `canActivate` 是异步函数（返回 `Promise<boolean>`），排序逻辑中需用 `await` 执行。
3. **优先级数值规则**：建议约定「数值越大优先级越高」，避免混乱；未标记优先级的守卫默认值设为 `0`。
4. **与局部守卫的配合**：全局守卫执行完毕后，才会执行控制器 / 路由上通过 `@UseGuards()` 注册的局部守卫。

## 总结

通过 `@SetMetadata()` 标记优先级元数据 + `Reflector` 读取排序，可突破 NestJS 全局守卫「注册顺序即执行顺序」的限制，实现：

- 全局统一的守卫优先级规则；
- 路由级别的优先级自定义；
- 异步守卫的有序执行。

这是大型项目中管理多全局守卫（如签名校验、登录态校验、权限校验、接口限流）的最佳实践，能确保守卫按业务逻辑的依赖顺序执行。
