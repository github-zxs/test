### 一、双系统启动完整流程
生成后的流程图会清晰呈现：UEFI → ESP（`/boot/efi`）→ GRUB → 分别启动 Ubuntu（依赖 `/boot` 内核 /initramfs）和 Windows 的完整路径。

### 二、/boot 与 EFI 分区的文件结构示意图

plaintext

```
硬盘（GPT分区表）
├─ ESP分区（FAT32，挂载 /boot/efi）
│  └─ EFI/
│     ├─ ubuntu/grubx64.efi （Ubuntu GRUB 入口）
│     └─ Microsoft/Boot/bootmgfw.efi （Windows 引导入口）
└─ /boot 分区（ext4，独立分区推荐1G）
   ├─ vmlinuz-6.5.0-28-generic （Linux内核）
   ├─ initramfs-6.5.0-28-generic.img （初始化内存镜像）
   └─ grub/ （GRUB 配置文件 grub.cfg 等）
```

# 一、启动流程 + 故障点 + 修复命令（最强版）

plaintext

```
开机
  ↓
UEFI 自检
  ↓
读取 EFI 分区（/boot/efi）→ 找引导文件
  ↓ 【故障：找不到引导 / 直接进Win / 直接进Ubuntu】
  ↓ 修复：BIOS 调整启动顺序，把 Ubuntu/GRUB 放第一
  ↓
启动 GRUB 菜单
  ↓ 【故障：没有 GRUB，黑屏】
  ↓ 修复：Ubuntu U 盘救援 → 重装 GRUB
  ↓
选择系统：
  ├─ 选 Ubuntu → 读 /boot 里的 内核 + initramfs
  │     ↓ 【故障：报错、进不去、内核panic】
  │     ↓ 修复：LiveCD 挂载 /boot，重新生成 initramfs
  │     ↓ 或：检查 /boot 是否满了
  │
  └─ 选 Windows → 跳转到 Windows 自己的 EFI 引导
        ↓ 【故障：选了Win进不去】
        ↓ 修复1：Ubuntu 里执行 sudo update-grub
        ↓ 修复2：Win 安装盘修复 EFI 引导
```

# 二、你最常用的 5 条救命命令（直接背）

## 1. GRUB 里看不到 Windows（最常见）

进 Ubuntu 执行：

bash

运行

```
sudo update-grub
```

## 2. GRUB 直接没了，进不了双菜单

用 Ubuntu U 盘试用模式执行：

bash

运行

```
sudo mount /dev/nvme0n1p5 /mnt
sudo mount /dev/nvme0n1p1 /mnt/boot/efi
sudo mount --bind /dev /mnt/dev
sudo chroot /mnt
grub-install /dev/nvme0n1
update-grub
exit
reboot
```

## 3. Windows 进不去，修复 Windows 引导

用 Win11 安装 U 盘 → 命令提示符：

cmd

```
diskpart
select disk 0
select partition 1
assign letter S
exit
bcdboot C:\Windows /s S: /f UEFI
```

## 4. /boot 太满导致无法更新 / 开机

bash

运行

```
# 查看旧内核
dpkg --list | grep linux-image

# 删除旧内核（替换成你不用的版本）
sudo apt remove linux-image-6.5.0-xx-generic
```

## 5. Win 和 Ubuntu 时间差 8 小时

bash

运行

```
timedatectl set-local-rtc 1 --adjust-system-clock
```

# 三、一句话记住所有逻辑

- **EFI = 引导大门**
- **GRUB = 双系统菜单**
- **/boot = Ubuntu 启动本体（内核 + 驱动）**
- 菜单坏了 → 修 GRUB
- Windows 进不去 → 修 Windows EFI
- Ubuntu 起不来 → 看 /boot
